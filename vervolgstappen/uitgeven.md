# Uitgeven

## Beschrijving

Als uitgever ben je verantwoordelijk voor het (langtermijn) beschikbaar maken van de code en bijhorende documentatie.  

Je bent een centrale aanspreek punt en faciliteert de communicatie tussen de partijen die aan, en met, de code en documentatie werken.  

Je overziet:
* de repository waar samengewerkt wordt
* de centrale issue en bug tracker
* de samenwerking tussen bvb ontwikkel- en review-partijen
* het bundelen van patches versies en daarbijhorende documentatie
* de communicatie van change logs, release dates, breaking changes, critical issues naar gebruikers van de code
* de qualiteit van installatie-, beheers, en gebruiks-handleiding

Je bent niet vantwoordelijk voor de code zelf, noch het aanmaken van documentatie.
Je houd wel het kwaliteits-niveau van informatie op peil, en stuur andere marktpartijen aan.
